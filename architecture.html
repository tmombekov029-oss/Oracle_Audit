Архитектура системы аудита Oracle

Понимание архитектуры аудита необходимо для эффективной настройки и управления системой безопасности базы данных.

Компоненты архитектуры

  • События базы данных - DML, DDL, сессии
  • Audit Trail - UNIFIED_AUDIT_TRAIL / DBA_AUDIT_TRAIL
  • Политики аудита - правила и условия аудита
  • FGA — детальный аудит на уровне строк
  • Управление и архивирование - жизненный цикл данных

Типы аудита

Тип аудита Хранилище Производительность Гибкость
Unified Auditing UNIFIED_AUDIT_TRAIL Высокая Очень высокая
Classic Auditing AUD$ Средняя Ограниченная
Fine-Grained Auditing FGA_LOG$ Зависит от условий Максимальная

Поток данных аудита


-- Создание события аудита
1. Пользователь выполняет операцию
2. Ядро Oracle проверяет политики аудита
3. Если условие выполнено - запись в буфер аудита
4. Периодическая запись из буфера в таблицы аудита
5. Доступ через системные представления

Проверка архитектуры


-- Проверка режима аудита
SELECT parameter, value 
FROM v$option 
WHERE parameter = 'Unified Auditing';

-- Просмотр политик
SELECT policy_name, enabled, audit_option
FROM audit_unified_policies
ORDER BY policy_name;

-- Статистика использования
SELECT audit_type, count(*)
FROM unified_audit_trail
GROUP BY audit_type;

Производительность


-- Мониторинг нагрузки от аудита
SELECT name, value
FROM v$sysstat
WHERE name LIKE '%audit%';

-- Размер таблиц аудита
SELECT table_name, num_rows, blocks
FROM dba_tables
WHERE table_name LIKE '%AUDIT%' OR table_name LIKE '%FGA%';
